# Инструментарий для реверс-инжиниринга Glyphix OS

Данный репозиторий содержит спецификацию и утилиты для исследования и распаковки пакетов `.pkg` операционной системы **Glyphix Framework**, используемой на смарт-часах (например, DT G1 на базе Actions Semiconductor ATS3085S).

## Содержимое репозитория

* **`PKG Specification.md`** — Подробная спецификация формата потоковых некомпрессированных архивов `.pkg`, включая описание глобального заголовка, таблицы смещений, путей файлов и форматов полезной нагрузки (включая AOT-скомпилированный JS байткод, UI-байткод и изображения PGF).
* **`un-pkg.py`** — Python-скрипт для распаковки `.pkg` архивов. Читает структуру архива, разбирает глобальный заголовок, таблицы файлов и имен, извлекает и сохраняет файлы с сохранением структуры директорий.
* **`js-strings.py`** — Утилита для извлечения пула констант (строк) из скомпилированного байткода JavaScript (`*.js` файлов) Glyphix OS. Включает анализ длины строк (Bitwise Length Encoding) и останавливается на маркере входа ВМ.

## Использование

### Распаковка `.pkg` файла

Скрипт `un-pkg.py` содержит функцию `unpack_pkg(pkg_path, out_dir)`, которую можно использовать для распаковки файлов.
Пример:
```python
from un_pkg import unpack_pkg

unpack_pkg("app.pkg", "./output_directory")
```

### Извлечение строк из скомпилированного JS

Скрипт `js-strings.py` содержит функцию `dump_js_bytecode_strings(js_filepath)`, которая позволяет парсить и выводить строки, зашифрованные в скомпилированном JS-файле.
Пример:
```python
from js_strings import dump_js_bytecode_strings

dump_js_bytecode_strings("./output_directory/main.js")
```
